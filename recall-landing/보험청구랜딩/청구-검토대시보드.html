이 대시보드 사용법

이 페이지는 청구 프로그램이 제안한 보험청구가 "이렇게 청구해도 되는지"를 사람이 다시 체크하는 검토용 화면입니다. 프로그램이 어떤 근거로 → 왜 → 어떤 청구를 했는지를 한눈에 보고, 확정/수정/보류만 누르면 됩니다.

🖥️ 1. 화면을 어떻게 띄우나

  • 왼쪽 절반엔 덴트웹, 오른쪽 절반에 이 창을 둡니다. (윈도우 ⊞+→ 우측 도킹)
  • 덴트웹에서 보는 환자의 차트번호를 이 페이지 큐(왼쪽 목록)의 파란 번호와 맞춥니다.

📋 2. 어떤 순서로 검토하나

  1. 상단 요약바로 오늘 전체 상황 파악 — 누락발견(받을 돈), 삭감회피(잃을 뻔한 돈), 비급여 전환 건수.
  2. 왼쪽 큐에서 환자를 위→아래로. 색이 위험 신호입니다.
  3. 오른쪽 상세에서 차트 원문 ↔ 제안 청구를 좌우로 대조. 같은 번호 1 끼리 근거↔결론이 연결됩니다.
  4. 아래 「왜 이 청구를 했나」 근거 카드에서 근거(차트 원문) → 규칙(P-XX) → 결론 → 신뢰도를 확인.
  5. 맞으면 확정, 고쳤으면 수정함, 애매하면 보류(2차 검토 큐로).

🎨 3. 색·배지 뜻 (triage)

정상 — 그대로 청구 OK 삭감위험 — 수정 필요 누락발견 — 청구 추가 가능 비급여/보류 — 정정 필요

제안 청구 라인의 배지: 정정 금액·코드 고침 · 추가 누락분 추가 · 삭감위험 그대로 내면 깎임 · 비급여 급여 불가 → 비급여 전환.

⌨️ 4. 키보드 (마우스 없이)

  • J/↓ 다음 환자 · K/↑ 이전 환자
  • 1 확정 · 2 수정함 · 3 보류 (누르면 자동으로 다음 환자)
  • ? 또는 / 이 사용법 열기 · Esc 닫기
⚠️ 마지막 책임은 사람입니다. 신뢰도(%)는 참고용 신호일 뿐, 고액·임플란트·비급여 전환은 신뢰도가 높아도 반드시 직접 확인하세요. 「고신뢰 일괄확정」은 정상(초록) 건만 묶어주지만, 누르기 전 목록을 한 번 펼쳐 봅니다.
