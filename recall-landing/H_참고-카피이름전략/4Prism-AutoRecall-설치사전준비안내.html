설치 사전 준비 안내

병원에 미리 전달하는 "설치 전 준비물" 한 장. 이대로 준비되어 있으면 설치 당일 원격 접속만으로 끝납니다.

대상 도입 치과 담당자(실장·접수) 버전 v1.0 · 2026-06-17 소요 병원 준비 10분 · 설치 30~60분

📌 가장 중요한 3가지 (이것만 지키면 90% 해결)

1. 서버 PC를 끄지 마세요. 설치 예정일 전날 밤부터 켜둔 상태로 두셔야 원격 접속이 됩니다.

2. 팀뷰어(TeamViewer) ID·비밀번호를 정확히 알려주세요. (서버 끄면 ID가 바뀌어 접속 불가)

3. 인터넷 전화 정보(통신사·번호) 또는 전화 단말기 모델명을 알려주세요.

최근 도입 과정에서 "서버를 끄고 퇴근해 원격 접속이 막혀 설치가 하루 밀린" 일이 있었습니다. 이 안내문은 그런 일이 다시 없도록 병원이 설치 전에 준비할 것만 모았습니다. 어려운 기술 용어는 접힌 박스 안에 따로 넣어 두었으니, 윗부분 체크리스트만 보셔도 됩니다.

1설치 전 체크리스트

설치 약속일 하루 전까지 아래 항목을 준비해 주세요. 필수 표시는 없으면 설치가 진행되지 않습니다.

  • 서버 PC 항상 켜두기 필수
    덴트웹(dentweb) 데이터가 있는 메인 PC를 설치 전날 밤부터 설치 완료까지 절전·종료하지 않기. 절전모드도 해제해 주세요.
  • TeamViewer 설치 + 정확한 접속정보 필수
    원격 설치용. ID와 비밀번호를 신청서에 기재. PC를 껐다 켜면 정보가 바뀌니, 알려주신 뒤에는 재부팅하지 말아 주세요.
  • 덴트웹 DB 접속 정보 필수
    설치 시 입력 창에 넣을 DB 정보(서버 주소·계정). 모르시면 덴트웹 담당자에게 미리 문의 부탁드립니다. 설치 마법사가 안내합니다.
  • 인터넷 전화 정보 또는 전화기 모델명 필수
    콜 자동기록용. ① 통신사·전화번호·OpenAPI 계정, 또는 ② 사용 중인 전화 단말기 모델명(예: IP520G). 둘 중 하나면 됩니다. (아래 4번 참고)
  • 접수 데스크 PC 위치 확인 권장
    리콜 대상 환자가 실제로 뜨는 화면이라, 어느 데스크 PC에 설치할지 미리 정해 주세요. (좌측/우측 등)
  • 원격 작업 가능 시간대 공유 권장
    진료시간 외 작업이 편하면 가능한 요일·시간을 알려주세요. (예: 진료 마감 후)

3설치 당일 진행 순서

아래는 담당자(4Prism)가 원격으로 진행하는 순서입니다. 병원은 PC를 켜둔 채 자리만 지켜 주시면 됩니다.

  1. 원격 접속 (TeamViewer)
    알려주신 ID·비밀번호로 접속합니다.
  2. 설치 프로그램 실행
    AutoRecall 설치 마법사를 실행합니다. (안내 문구가 단계별로 표시됩니다)
  3. 덴트웹 DB 정보 입력
    초기 설치 창에 DB 접속 정보를 입력합니다.
  4. 전화 연동 설정
    통신사 API 또는 전화 단말 신호 방식으로 콜 기록을 연결합니다. (4번 참고)
  5. 리콜 대상 화면 확인
    접수 데스크에 어떤 환자가 리콜 대상으로 뜨는지 함께 확인합니다.
  6. 개선 참여 동의 확인
    설정 페이지의 "프로그램 개선 참여" 체크 여부를 확인합니다. (6번 참고)

4전화 연동 방식 (둘 중 하나)

콜(전화) 기록을 자동으로 가져오는 방법은 병원 환경에 따라 두 가지입니다. 병원은 현재 쓰는 통신사·전화기만 알려 주시면, 어느 방식이 맞는지 저희가 판단합니다.

방식필요한 것장점 / 한계
① 통신사 API통신사·번호·OpenAPI 계정장치 추가 없음 · 통신사 지원 필요
② 전화 신호 수집전화 단말기 + 소형 네트워크 장치통신사 무관·범용 · 장치 설치 필요
🔧 (기술 참고) 전화 신호 수집 방식 상세 — 담당자용

인터넷 전화는 보통 두 계층으로 동작합니다.

  • SIP (시그널) — 누가 누구에게 거는가, 응답·종료 시각. (포트 5060/5061)
  • RTP/SRTP (음성) — 실제 통화 음성. (동적 포트)

콜 발/수신 시각·번호만 필요하므로 SIP 시그널만 수집하면 됩니다. 음성(RTP)은 가져오지 않습니다 → 통화 도청 우려 없음.

전화 단말 자체에는 기록이 저장되지 않고 스위치에서 신호를 당겨오는 구조라, 보통 공유기(예: 192.168.219.1) ↔ 전화 단말 사이에 미러링 장치(예: TL-SG105E)를 끼워 SIP 신호만 복사해 수집합니다.

검토 중

SIP 시그널을 암호화 이슈 없이 가져올 수 있는지 테스트 진행 중(사계절치과 IP520G 단말 기준). 범용성 확보를 위해 전화기 포함 패키지 판매도 옵션으로 검토.

5자주 생긴 문제와 예방

실제 도입 과정에서 나온 문제들입니다. 미리 알면 대부분 피할 수 있습니다.

증상원인예방
원격 접속 불가서버를 끄고 퇴근 → 팀뷰어 ID 변경설치 전날부터 PC 켜두기·절전 해제
설치 중 DB 연결 실패덴트웹 DB 정보 미확보사전에 덴트웹 담당자에게 정보 확인
콜 기록이 안 들어옴통신사 API 미지원·전화기 정보 미상전화기 모델명을 사전에 공유
화면이 외부로 넘어감리콜 페이지에서 병원 홈페이지로 이동외부 링크 차단된 전용 화면으로 제공(적용됨)

6로그 수집 · 개선 참여 동의

프로그램을 더 좋게 개선하기 위해, 사용 중 발생하는 화면 이동·버튼 클릭 같은 사용 기록을 수집합니다.

  • 설정 페이지의 "프로그램 개선 참여" 체크박스로 켜고 끌 수 있습니다. (기본 켜짐)
  • 수집 항목: 어떤 화면을 봤는지, 어떤 버튼을 눌렀는지, 오류 발생 여부.
  • 환자 진료정보·통화 음성은 수집하지 않습니다.

✅ 동의는 병원이 직접 선택

심도 있는 로그 수집은 개선 프로그램 참여 동의를 받은 경우에만 진행합니다. 설치 시 동의 여부를 함께 확인합니다.
