접수 · 진료중 · 진료완료 — 세 번 잡는 3중 차단

한 번 알림은 바쁘면 지나칩니다. 그래서 환자 상태가 바뀔 때마다 다시 띄우고, 마지막 관문(진료완료)에서는 청구가 안 돼 있으면 넘어가지지 않습니다.

바로 청구하고 완료사유 남기고 완료

진료완료 버튼을 누르는 순간이 청구의 골든타임 — 지나가면 아무도 다시 열어보지 않으니까, 여기서 막습니다.

🔁 데스크 → 진료실 릴레이

데스크에서 환자가 수락하면 버튼 한 번으로 차트 메모(“환자분 요청으로 진행”)가 남아 진료실은 무조건 진행. 데스크가 애매하면 부담 없이 패스 — 진료 시작 때 진료실 팝업이 사진 보여주며 말할 설명 전략과 함께 이어받습니다. 둘 중 하나만 성공해도 청구가 됩니다.

화면·멘트는 실제 제품 흐름 기준 예시입니다. 진료 필요 여부 판단과 청구 승인은 원장님(담당 직원)이 합니다.

사람 문제까지

전 직원이 청구 전문가일 수는 없습니다.
시스템이 받치고, 리포트가 알아봐 줍니다

“무료가 어떻게 가능해요?” → 이미 하신 진료의 돈을 찾아드리는 것이라, 앱이 100만원을 먼저 벌어드린 뒤에만 전환을 안내합니다.

비교는 일반적 운영 형태 기준이며 업체별 조건은 다를 수 있습니다.

❓ FAQ

Q기존 청구 방식을 바꿔야 하나요?
아니요. 지금 방식 그대로 하시고, 빠진 것만 알림으로 잡아드립니다.
Q덴트웹이 아니어도 되나요?
현재 덴트웹 연동 기준입니다. 타 차트는 도입 문의 시 일정을 안내드립니다.
Q알림이 진료를 방해하지 않나요?
환자 상태가 바뀌는 순간(접수·진료중·완료)에만 뜹니다. 하루 평균 5~7건 수준입니다.
Q무료 조건이 정확히 뭔가요?
앱이 만들어 낸 누적 성과(청구 지원 100만원 또는 리콜 수익 100만원) 도달 전까지 무료. 도달 시 충분한 사전 안내 후 정식 전환을 선택하시게 됩니다.

🎁 도입과 무관하게 드리는 선물 — 무료 전자책 「빈틈없는 데스크」(데스크가 새는 돈을 막는 실전 워크북). 담당자에게 요청하시면 바로 보내드립니다.

무료·전환 조건 상세는 설치 화면과 약관에 동일하게 고지됩니다.
