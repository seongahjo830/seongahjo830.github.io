제작 리서치 · How to build it well

같은 스토리라인을 '완전히 다른 3방식'으로
만드는 이상적 방법

기존 스토리라인 설계도(3막 8장)는 그대로. 이번 리서치는 "그 서사를 어떤 형식·인터랙션·비주얼 훅으로 풀어야 전환이 오르나"에 대한 표준 조사 결론.

한 줄 결론(BLUF). ① 인터랙티브 자가진단(슬라이더·퀴즈)은 정적 폼 대비 전환 약 16배 — 3방식 전부의 심장으로 유지하되 "내 숫자를 먼저 보여주고" 연락처는 나중. ② 히어로는 5초 패턴 인터럽트 + 짧은 헤드라인 + CTA 1개 + 결과(아웃컴) 중심. ③ 모바일 우선·3초 내 로딩·외부 의존성 0. ④ 차별화는 장르(형식) 자체를 바꾸는 것 — 그래서 같은 8장을 터미널 콘솔 / 메신저 대화 / 파티클 데이터비주얼 세 장르로 분기.

1. 표준 리서치 결론 — '이상적'의 근거

원칙근거 수치/출처3방식 적용
인터랙티브 우선인터랙티브 폼/퀴즈 전환 40~47% vs 정적 2.8% (≈16x) 🟢자가진단 슬라이더를 세 장르 각각의 UI로 내장(터미널 입력·챗봇 질문·파티클 재점등)
가치 먼저(Value-first)"결과를 먼저 주고, 연락처는 게이트 뒤로" — 신뢰·데이터 동시 확보 🟢설치 전에 "내 숫자(잠든 환자·잃은 매출)"부터 화면에서 즉시 계산
히어로 5초 룰짧은 헤드라인·CTA 1개·미세 애니메이션·다중 의사결정자(원장+실장) 🟡세 장르 모두 첫 화면에 강한 모션 훅 + "안 부른 환자만" 한 문장
대화형의 힘대화형 경험 전환 3~4x·"들어준다"는 감각 🟢2번 방식(메신저)이 정면으로 활용 — 원장·실장 대화로 통증을 '엿보게'
테크니컬 모노모노스페이스·터미널 미감 = 통제·장인·진정성 신호(과장 광고 반대) 🟡1번 방식(콘솔)이 "운영 도구" 포지셔닝과 정확히 일치(효과 보장 X)
아웃컴 > 기능"쓰고 나면 뭐가 달라지나"를 보여주기 🟡케이스 좌(현실)/우(회수)·벤치마크로 변화 가시화
모바일·속도B2B 구매자 80%+가 폰으로 조사·3초 내 로딩 🟢반응형·단일 HTML·외부 의존성 0(폰트 1개 CDN, 폴백 시스템폰트)

2. 결정 — '전혀 다른 3방식'은 장르를 바꾼다

색만 바꾸면 '다른 방식'이 아니다. 읽는 사람의 행동 양식(인터랙션 메타포) 자체를 바꿔야 진짜 다른 경험. 같은 3막 8장 + 같은 자가진단을 세 장르로:

방식 1 · CONSOLE

터미널 진단 콘솔

차트를 스캔하는 시스템처럼. 타이핑 부팅 → "347명 발견" 경고 로그 → 명령으로 자가진단. 장인·통제·진정성 신호. '운영 도구' 포지셔닝과 일치.

방식 2 · MESSENGER

원장·실장 대화 엿보기

메신저 화면처럼. 두 사람의 대화로 통증·면죄가 자연스럽게 드러남 → 진단봇이 끼어들어 '내 숫자' 계산. 대화형 3~4x·공감.

방식 3 · PARTICLE

잠든 환자 347개의 빛

환자 한 명 = 빛나는 점 하나. 스크롤하면 점들이 꺼져가고(잊혀짐), 자가진단으로 '내 환자'가 다시 켜진다. 이중부호화·보유효과로 각인.

3. 가드레일 (세 방식 공통, 반드시)

  • 의료효과·치료결과 보장 표현 금지 — "전화 우선순위 정리를 돕는 운영 도구"로만.
  • 모든 수치 = 예시·가정값 명시 + 계산식 공개(슬라이더 위/아래에 항상 노출).
  • 입력값 저장·전송 안 함 — 화면 안에서만 계산. 환자정보 원내 로컬·차트 읽기만.
  • 한글 우선, 영어는 괄호 병기. 반응형·외부 의존성 최소.
