제안 설계도 · 3 Storyline Approaches

같은 설계도, 전혀 다른 3가지 방식

기존 스토리라인 설계도(3막 8장 + 자가진단)의 내용·수치·전환장치는 그대로 두고, 장르·시각언어·감정선만 완전히 다르게 푼 세 가지 랜딩. “먼저 방식을 제안하고, 그 방식대로 만든다”의 결과물.

한 줄(BLUF). 세 방식 모두 같은 8장 서사(히어로→통증→증폭→면죄→케이스→자가진단→벤치마크→설치)와 같은 계산식(잠든환자=전체×끊김율 · 잃은매출=잠든×객단가 · 회수=×0.15 · 점수=(1−끊김율)×100)을 쓴다. 다른 건 “어떻게 보여주느냐” 하나뿐 — ① 느와르 추리극(사건·증거·도장) ② 키네틱 영수증(실시간 인쇄·합계 틱업) ③ 스크롤 시네마(09:00→마감, 빛이 바뀌는 하루).

세 가지 방식

🔦사건 파일 No.347느와르 추리극
사라진 환자 = 미결 실종 사건, 원장 = 형사. 어두운 수사보드 + 붉은 실로 이어진 흐릿한 증거 카드 + “미결(未決)” 도장으로 5초 안에 “내 얘기”로 끌어들인다.
첫 훅수사 보드 + 붉은 실 + 타자기 헤드라인 “사라진 건 환자가 아니라 당신의 매출”
자가진단“사건 파일 열기” — 슬라이더=수사 입력, 결과=증거 도시에(Dossier)로 도장
감정선긴장·호기심 → 무혐의(면죄) → 종결(행동)
→ 1번 방식 열기
🧾안 부른 환자 정산서키네틱 영수증
페이지 전체가 감열프린터 영수증. 로드되자마자 한 줄씩 인쇄되고, “이번 달 놓친 매출” 빨간 숫자가 실시간으로 틱업된다. 손실회피를 가장 물리적으로 보여주는 방식.
첫 훅톱니 가장자리 영수증이 인쇄되는 애니메이션 + ₩ 카운터 상승
자가진단슬라이더를 움직이면 영수증 합계가 그 자리에서 재인쇄(영수증=결과패널)
감정선충격(적자) → 면죄(과실 아님) → 정산 마감(CLOSED 도장)
→ 2번 방식 열기
🎬데스크의 하루스크롤 시네마
실장 1인칭으로 09:00 개원부터 마감까지 따라가는 시네마틱 스크롤. 스크롤하면 하늘빛이 아침→한낮→노을→밤으로 바뀌고, “못 부른 명단”이 데스크에 쌓인다. 공감 우선.
첫 훅풀블리드 + 큰 시계 09:00 + 울리는 전화·쌓이는 서류(순수 CSS 애니메이션)
자가진단마감(19:00) 불 꺼진 어둠 화면에서 “오늘 못 부른 사람 몇 명?” 슬라이더
감정선몰입(하루) → 연민(짧은 하루) → 여명(내일은 다르게)
→ 3번 방식 열기

왜 “세 방식”인가 — 자기참조를 거는 각도가 다르다

같은 사람이라도 끌리는 입구가 다르다. 셋은 self-relevance(내 얘기)를 거는 각도를 일부러 분산했다.

방식핵심 심리 레버가장 센 순간
🔦 추리극호기심 갭 · 미스터리 · 패턴 인터럽트“미결 도장”과 붉은 실 — 장르 전환으로 광고 회피 뚫기
🧾 영수증손실회피(Kahneman) · 구체성 · 즉시성₩ 카운터가 오르는 1.9초 + 합계 실시간 재인쇄
🎬 시네마공감 · 서사 몰입(Green/Brock) · 책임 재귀인불 꺼진 마감 화면에서 “오늘 못 부른 몇 명?”

공통 가드레일(셋 다 지킴)

  • 의료효과·치료결과 보장 표현 금지 — “전화 우선순위 정리를 돕는 운영 도구”로 포지셔닝.
  • 모든 수치는 예시·가정값 + 계산식 공개 · 입력값 저장·전송 안 함(화면 내 계산).
  • 반응형 · 외부 의존성 0 · 한글 우선(영어 괄호 병기).
